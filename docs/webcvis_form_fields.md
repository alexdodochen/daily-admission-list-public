# WEBCVIS HCO1W001 Cathlab Schedule Page - Playwright Automation Reference

This document provides a detailed reference for the form fields, buttons, and table structure of the `HCO1W001.do` page, specifically tailored for Playwright automation.

## 1. Form HCO1WForm

*   **Action URL**: `/WEBCVIS/HCO/HCO1W001.do`
*   **Method**: `post`

### Hidden Inputs

The form contains several hidden input fields, including a Struts CSRF token.

| Name                        | ID                      | Default Value                                | Purpose                                   |
| :-------------------------- | :---------------------- | :------------------------------------------- | :---------------------------------------- |
| `org.apache.struts.taglib.html.TOKEN` | (none)                  | `f3a96086cf40f80c9e49e44949dcef07` (example) | Struts CSRF token                         |
| `pageid`                    | `pageid`                | `HCO1W001`                                   | Page identifier                           |
| `buttonName`                | `buttonName`            | (empty)                                      | Used by action buttons to specify operation |
| `pageSize`                  | `pageSize`              | `9999`                                       | Pagination setting                        |
| `index`                     | `index`                 | `0`                                          | Row index for operations                  |
| `hesid`                     | `hesid`                 | (empty)                                      | Hidden ID, possibly for selected record   |
| `hctjson`                   | `hctjson`               | (empty)                                      | JSON array for Heart Catheter Type        |
| `pdijson`                   | `pdijson`               | (empty)                                      | JSON array for Pre-Diagnosis Item         |
| `phcjson`                   | `phcjson`               | (empty)                                      | JSON array for Pre-Heart Catheter         |
| `adijson`                   | `adijson`               | (empty)                                      | JSON array for After Diagnosis Item       |
| `adiContentTx`              | `adiContentTx`          | (empty)                                      | After Diagnosis Content                   |
| `finishjson`                | `finishjson`            | (empty)                                      | JSON array for finish status              |
| `state`                     | `state`                 | (empty)                                      | Current state of an operation             |
| `refernojson`               | `refernojson`           | (empty)                                      | JSON array for referral numbers           |
| (no `name` attribute)       | `adiFlag`               | (empty)                                      | After Diagnosis Flag                      |

## 2. Form fields by purpose

### Patient Identity

| Field Name  | Selector                        | Input Type | Notes                                                                                                        |
| :---------- | :------------------------------ | :--------- | :----------------------------------------------------------------------------------------------------------- |
| `patno2`    | `input[name="patno2"]`          | `text`     | Read-only when row selected. `keydown` event on Enter triggers AJAX call to `/WEBCVIS/ajax/AJAXAction.do` to populate patient info. |
| `patname`   | `input[name="patname"]`         | `text`     | Read-only when row selected.                                                                                 |
| `bedno`     | `input[name="bedno"]`           | `text`     |                                                                                                              |
| `patbirthday` | `input[name="patbirthday"]`     | `text`     | Read-only when row selected.                                                                                 |
| `patgender` | `select[name="patgender"]`      | `select`   | The `select` element can be hidden, with a `div id="displaytxt"` showing the gender text. Read-only when row selected. |

### Schedule Slot

| Field Name     | Selector                         | Input Type | Notes                                                         |
| :------------- | :------------------------------- | :--------- | :------------------------------------------------------------ |
| `examroom`     | `select[name="examroom"]`        | `select`   |                                                               |
| `inspectiondate` | `input[name="inspectiondate"]` | `text`     | Associated with a calendar picker. Value needs to be set manually or via calendar. |
| `inspectiontime` | `input[name="inspectiontime"]` | `text`     |                                                               |

### Diagnosis/Procedure

| Field Name           | Selector                               | Input Type      | Notes                                                              |
| :------------------- | :------------------------------------- | :-------------- | :----------------------------------------------------------------- |
| `prediagnosisitem`   | `textarea[name="prediagnosisitem"]`    | `textarea`      | Read-only. Has a `...` button (`#pdibt`) to open popup `HCO1N002.do`. |
| `preheartcatheter`   | `textarea[name="preheartcatheter"]`    | `textarea`      | Read-only. Has a `...` button (`#phcbt`) to open popup `HCO1N004.do`. |
| `pdijson`            | `input[name="pdijson"][type="hidden"]` | `hidden`        | JSON array for pre-diagnosis data.                                 |
| `phcjson`            | `input[name="phcjson"][type="hidden"]` | `hidden`        | JSON array for pre-heart catheter data.                            |
| `adijson`            | `input[name="adijson"][type="hidden"]` | `hidden`        | JSON array for after diagnosis data.                               |
| `hctjson`            | `input[name="hctjson"][type="hidden"]` | `hidden`        | JSON array for heart catheter type data.                           |
| `finishjson`         | `input[name="finishjson"][type="hidden"]` | `hidden`        | JSON array used for saving "finish" status.                        |
| `refernojson`        | `input[name="refernojson"][type="hidden"]` | `hidden`        | JSON array for referral numbers.                                   |
| `afterdiagnosisitem` | `textarea[name="afterdiagnosisitem"]`  | `textarea`      | Has a `...` button (`#adiNewBt`) to open popup `HCO1N_AfterDiagnosis.jsp`. |

### Note/Status

| Field Name     | Selector                                 | Input Type | Notes                                      |
| :------------- | :--------------------------------------- | :--------- | :----------------------------------------- |
| `note`         | `input[name="note"]`                     | `text`     |                                            |
| `state`        | `input[name="state"][type="hidden"]`     | `hidden`   | Used to track operation state.             |
| `adiContentTx` | `input[name="adiContentTx"][type="hidden"]` | `hidden`   | Content for After Diagnosis, likely from popup. |
| `adiFlag`      | `input[id="adiFlag"][type="hidden"]`     | `hidden`   | Flag for After Diagnosis, only has an ID.  |
| `casestate`    | (within table row)                       | `hidden`   | Status within a table row, updated by user. |
| `finishflgstate` | (within table row)                       | `hidden`   | Original finish status of a table row.     |

## 5. Row click handler (`$("#row tr:not(:has(th))").click`)

The `addRowHandlers` JavaScript function attaches a click event listener to each non-header row of the `#row` table.

### Pseudo-code

```javascript
// Inside addRowHandlers function for each row click event:
$(this).click(function(event) {
    // Prevent action if checkbox, select, or option within the row is clicked

        document.getElementsByName("examroom")[0].value = $(this).find("#hes_mcrid").val();

## 6. DEL flow

The verified deletion flow involves:

1.  Selecting one or more rows by checking the individual checkboxes (`input[name="chk"]`) present in each data row of the table.
2.  Clicking the main `deleteButton` (`id="deleteButton"`).
3.  A confirmation dialog (`deleteMsg()`) appears asking "您確定要刪除嗎？" (Are you sure you want to delete?).
4.  If the user confirms (clicks "OK"), the form is submitted with `buttonName` set to "DEL".

There is no explicit `checkedShowButton` in the provided HTML. The deletion mechanism is a global action triggered by `deleteButton` after row selection via checkboxes.

## 8. Date field gotchas

The date input fields `daySelect1` and `daySelect2` are initially rendered with the `readonly` attribute:

```html
<input type="text" name="daySelect1" id="daySelect1" readonly="" size="15" maxlength="60" value="2026/04/04">
<input type="text" name="daySelect2" id="daySelect2" readonly="" size="15" maxlength="60" value="2026/04/04">
```

Although they have calendar popups (`showCalendar` function calls) to assist in setting values, direct text input is prevented by the `readonly` attribute. Playwright automation would need to either:
1.  Remove the `readonly` attribute using `page.evaluate()` before attempting to type into the field.
2.  Interact with the calendar popup directly, if feasible.

This read-only status is explicitly ignored in the `allclearid()` function (`Ignoreid=["daySelect1","daySelect2"]`), indicating they are not cleared with other text fields, likely due to their date picker functionality.
