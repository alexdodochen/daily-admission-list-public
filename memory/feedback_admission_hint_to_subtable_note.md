name: 入院提示文字註記要搬到子表格 H 欄
description: 主資料 K 欄（入院提示）若有非純數字的注意事項（無床延期、陪病確診、建議再入院日等），抽出來寫到子表格 H 欄（註記）
type: feedback
---

主資料 K 欄（入院提示）內容分兩類：
- **純數字 / 數字組合**（如 `1`, `2`, `23`, `3.2`, `2.3.1`）= 住服優先級代碼，**不搬**
- **文字注意事項** = 重要備註，**抽出來寫到子表格 H 欄（註記）**。常見 pattern：
  - `X月X號改期` / `5/X 改期` / `延後到 5/X`
  - `建議再入院日 YYYY/MM/DD`
  - `X月X號無床延期` / `無床`
  - `(陪病 X 月初確診)` / 陪病感染史
  - `要外科會診` / `要 anesth 會診`
  - 任何中文敘述句

數字+文字混合（如 `3 建議再入院日2026/05/15`、`2.3.1(陪病3月初確診)`）：保留 K 欄原樣不動，只把文字部分抽到 H 欄。

**Why:** 5/4 user 主動提：「截圖中入院提示裡面有 x月x號無床延期等重要注意事項要幫我放入 subtable 備註」。這些文字代表病人有特殊狀況（無床、延期、陪病感染史等），下游 ordering 寫 N-V 時 H 會被讀進 R 欄（備註），住服 / 主治醫師看到才知道；只放在主資料 K 不會被下游 pipeline 用到。

**How to apply:**
- 每次跑 admission-image-to-excel 寫主資料 A-L 時，掃 K 欄
- 用 regex 抽出非數字 / 非標點的中文文字部分（建議：`re.sub(r'^[\d\.\s]+', '', k_value).strip()` 後若非空就視為文字註記）
- 寫到對應病人在子表格的 H 欄（B 欄病歷號比對）
- 主資料 K 欄保留原樣（不動）—— 住服仍要看完整版
- EMR 階段如果 H 已有值，**不要覆蓋**（process_emr 只在 H 為空時才寫「無資料病人」之類的標記）
