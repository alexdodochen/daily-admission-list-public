description: 抽籤決定醫師順序，入院序用真正的round-robin（輪流一人一人），非時段醫師最後才加入
type: feedback
---

**Round-Robin 規則（之前實作錯誤，已修正）：**
- 抽籤決定醫師順序（如 A→B→C）
- 入院序：A第1人 → B第1人 → C第1人 → A第2人 → B第2人 → C第2人...
- **不是** A全部 → B全部 → C全部（那是錯的）
- 各醫師內部順序依 E 欄（手動設定入院序）決定，無E欄的按原始順序

**抽籤流程：**

**A. 預設（無使用者指定時）：**
1. 時段醫師組 RR：ABCABCABC（一輪一個病人，醫師順序由 random shuffle 決定 *2 兩支籤）
2. 非時段醫師組 RR：接在時段組之後（序號接續），同樣 ABCABCABC pattern

**B. 使用者明確指定順位 → override 預設，可跨組：**
- 「黃鼎鈞 #1，黃睦翔 #2」= 第 1 位 黃鼎鈞、第 2 位 黃睦翔，**就算 黃睦翔 是非時段也照辦**
- 「其他照規則排列」= 沒指定的醫師按預設規則往後填（時段剩餘 → 非時段剩餘）
- 使用者也可指定「廖先詹後」之類的次序提示，覆蓋 random shuffle

**判斷指令是否為跨組 override 的關鍵：**
- 含明確序數（「#1」「第一順位」「#2」「第二」）→ 該醫師強制該位置（可跨組）
- 只說「然後是 X」（沒明確序數）→ 模糊，回去問或保守照預設規則

**Why（兩次踩雷紀錄）：**
2026-05-01 使用者說「黃鼎鈞放第一順位 然後是黃睦翔」（5/3 入院）。Claude 第一次解為跨組 #1+#2 → 黃睦翔 #2（非時段排在前面）；使用者糾正「非時段都要排在時段之後」 → Claude 修成兩段獨立 RR（黃睦翔 #7）；使用者再糾正「我5/3要黃鼎鈞第一 黃睦翔第二 其他照規則排列」 → 確認**明確序數就是跨組 override**。最終 5/3 N-V：黃鼎鈞、黃睦翔、廖瑀、詹世鴻、鄭朝允、黃鼎鈞、詹世鴻、詹世鴻。

**Why:** 之前錯把每位醫師的病人連排（block），應為輪流一人一人的round-robin。

**How to apply:** 抽籤時建立每位醫師的 patient queue，再用 while any(queues) 輪流取各醫師第一位病人。
