---
name: 入院清單變動更新流程（差異更新）
description: 同一日期的入院清單後續會變動，收到新截圖時要做差異比對，不能整張覆蓋
type: feedback
---

當 user 第一次給某日期（如 5/20）的入院清單截圖時，照常用 admission-image-to-excel 建立工作表。**但**該日期的入院清單在實際入院日之前還會變動，user 會在臨近日（如 5/17）再給一張新的 5/20 截圖。收到新截圖時必須做**差異更新**，而不是重建工作表。

**Why:** 整張覆蓋會弄丟已經做好的 EMR 摘要、user 手動拉的 F/G 下拉選單值、抽籤排好的入院順序、子表格的註記等。User 已花時間整理過的內容絕不能因為一張新截圖就清空。

**How to apply:**

收到「同一日期」的新截圖時：

1. **辨識新截圖** → 解析出新的病人清單（病歷號、姓名、主治醫師、診斷碼…）
2. **讀取現有 sheet** → 取得舊的 A-L 病人清單
3. **以病歷號為 key 做 diff**：
   - **新增病人**（在新截圖、不在 sheet）：列出來給 user 看
   - **取消病人**（在 sheet、不在新截圖）：列出來給 user 看
   - **保留病人**（兩邊都有）：保留原 row 不動
4. **取得 user 確認** → 確認新增/取消後才動手
5. **執行更新**：
   - 新增的病人：append 到主表 A-L 末尾、append 到對應醫師子表格底下、跑 EMR 摘要、預填 F/G dropdown
   - 取消的病人：從主表 A-L 刪除該 row、從醫師子表格刪除對應 row、從 N-W 入院序刪除
   - 保留的病人：**完全不動**（EMR 摘要、F/G 值、入院序、註記都保留）
6. **重新整理格式**：依照 `feedback_sheet_formatting.md` 規範套用統一格式（藍標題、白資料、邊框、靠左、F/G dropdown、空白行白底、欄寬）
7. **重新計算 N-W 入院序**：
   - 若 user 已手動設定過入院序（依 `feedback_preserve_manual_ordering.md`），詢問是否要重跑
   - 否則跑 round-robin：保留的病人沿用、新病人插入、取消的拿掉
8. **報告差異** → 列出本次新增 X 人、取消 Y 人、保留 Z 人，並提示 user 是否重跑抽籤/EMR/排程

**注意：**
- 主治醫師子表格的「人數」標題（如「劉嚴文（2人）」）要更新
- 若有病人換醫師（同病歷號但醫師不同），視為「舊醫師取消 + 新醫師新增」處理
- 若病人連同 cathlab 排程都要刪除，要在後續 cathlab keyin 時記得跳過或執行 DELETE
