---
name: 保留手動設定的入院序
description: 重新整理入院清單時，絕不可覆蓋使用者已手動設定好的 N-V 欄入院序
type: feedback
---

規則：若某日期 sheet 的 N-V 欄（序號｜主治醫師｜病人姓名｜備註(住服)｜備註｜病歷號｜術前診斷｜預計心導管｜每日續等清單）已有資料，視為使用者手動敲定的入院序，不可重跑 round-robin 覆寫。重新整理時須先讀取該區域，若非空則保留，只補空缺或做 read-only 的彙整輸出。

**Why:** 使用者常依臨床狀況（病人狀態、家屬、床位）手動調整 round-robin 的結果；Claude 若重跑 round-robin 會把這些調整沖掉，屬於不可逆的資料損失。

**How to apply:**
- 收到「整理 X 到 Y 的入院清單 / 產出各醫師清單」類指令時，先讀目標日期 sheet 的 N-V 欄
- 若 N-V 欄有資料：以該內容為準做彙整、分組、輸出，不碰 Sheet 寫入
- 若 N-V 欄為空：依照 Workflow 規則 13（等使用者確認 F/G 欄）再跑 ordering
- 任何情況下都不能用 round-robin 結果去覆蓋已填好的 N-V 欄
- 同樣適用於 A-L 主表與子表格中被使用者手動改過的欄位（姓名、病歷號、備註等）
