============================================
  第一階段：匯入與抽籤
============================================

--------------------------------------------
步驟一：匯入病人資料 [skill: admission-image-to-excel]
--------------------------------------------
  你的操作：
  - 直接將入院名單截圖貼給 Claude
  - 可一次貼多天的截圖
  - 範例指令：「這是 20260407-09 的三天預約入院清單，請幫我整理」

  Claude 會：
  1. 辨識圖片中的病人資料（12 欄：A-L）
  2. 自動比對主治醫師正確名單，修正 OCR 誤讀
  3. 自動建立日期工作表，寫入 A-L 欄
  4. 列出所有病歷號請你確認
     （OCR 讀取數字容易出錯，務必逐筆核對！）

  注意：
  - 截斷的姓名不需要補全，步驟三 EMR 會自動校正
  - 病歷號固定 8 碼，以文字存入（保留前導零）

--------------------------------------------
步驟二：主治醫師抽籤排序 [skill: admission-lottery]
--------------------------------------------
  你的操作：
  - 「抽籤」或「排入院順序」
  - 指定醫師優先：「抽籤，陳則瑋排第一」
  - 特殊排序：「無條件讓 XXXXXXXX 第一個入院，備註員工家屬」

  Claude 會：
  1. 判斷入院日 → 查隔天門診醫師（主治醫師抽籤表）
     日→一、一→二、二→三、三→四、四→五
     （五/六需你手動指定）
  2. 區分醫師：
     ─ 在抽籤表 → 參加主抽籤
     ─ 不在抽籤表 → 仍留在工作表，但排在主抽籤之後
  3. 隨機抽籤，*2 醫師兩支籤
  4. 產生 round-robin（N-P 欄：序號、主治醫師、病人姓名）
  5. 在主資料下方排列各醫師病人表格（8 欄 A-H）
     A=姓名  B=病歷號  C=EMR  D=EMR摘要
     E=手動設定入院序  F=術前診斷  G=預計心導管  H=註記
     ─ F、G 欄有下拉選單（與導管排程網站選項同步）

  【重要：非抽籤表醫師的處理】（已被修正多次，務必遵守！）
  - 所有醫師的病人都留在當日工作表，不會被移走
  - 不在抽籤表的醫師絕對不能放入主抽籤的 round-robin
  - 主抽籤所有有時段醫師的病人全部排完後：
    → Claude 先問你：「非時段醫師（如 XXX）的病人要不要
       跟每日續等清單合併抽籤？」
    → 等你回覆後才繼續排非時段醫師的入院序
    → 非時段醫師的序號接在有時段醫師最後一位病人之後

--------------------------------------------
步驟三：自動提取 EMR 病歷摘要 [skill: admission-emr-extraction]
--------------------------------------------
  前提：使用者手動在 Chrome 開啟 Web EMR 並登入（有 CAPTCHA，無法自動登入）
  登入後將 session URL 貼給 Claude（格式：(S(xxxxx))/tree/frame.aspx）
  或直接貼 EMR 截圖讓 Claude 從圖片做摘要

  Claude 會：
  1. 用 Playwright 帶 session URL 查詢各病人（搜尋框 ID: txtChartNo）
  2. 點進心臟科門診紀錄，擷取 SOAP note
  3. 自動校正姓名（EMR 為準，更新 Sheet 所有位置）
  4. 生成四段式摘要：
     一、心臟科相關診斷
     二、病史
     三、客觀檢查
     四、本次住院計畫
  5. 自動寫入醫師病人表格 C/D 欄（不需再問確認）
     C 欄=完整 EMR 原文（可收縮）
     D 欄=四段式摘要
  6. 無紀錄 → C 欄寫「無本院一年內主治醫師門診紀錄」

  技術補充：
  - EMR frame 結構：top.aspx → list3.aspx（就醫紀錄樹）→ content frame
  - 首次批次完成後會重試載入失敗的病人
  - 會檢測並修正重複資料（兩人拿到同一份 EMR）

  【重要】Claude 絕對不要自己開瀏覽器嘗試登入 EMR，等使用者手動登入後再操作


============================================
  第二階段：入院序整合
============================================

--------------------------------------------
步驟四：設定入院順序與下拉選單 [skill: admission-ordering]
--------------------------------------------
  【重要流程！Claude 必須遵守的順序】
  1. 抽籤 → 顯示結果（步驟二）
  2. 自動開始 EMR 摘要擷取（步驟三）
  3. EMR 完成後 → 等你手動在醫師病人表格的
     F 欄（術前診斷）和 G 欄（預計心導管）拉下拉選單
  4. 你確認完下拉選單後 → Claude 才一併把
     抽籤結果 + 術前診斷 + 預計心導管
     寫入 N-S 入院序列表格

  *** Claude 絕對不能在你拉完下拉選單之前寫入 N-S 欄 ***

  你的操作：
  - EMR 完成後，在 Excel 的醫師病人表格 F/G 欄選擇下拉選單
  - 如果病人不需要導管排程，F/G 欄留空即可
  - 選完後告知 Claude：「下拉選單填好了，幫我整合入院序」

  Claude 會：
  1. 再次確認所有醫師是否在隔天抽籤表上
  2. 如有 2 位以上病人的醫師，列出供你決定順序
  3. 你確認後，一次寫入 N-T 欄（7 欄）：
     N=序號  O=主治醫師  P=病人姓名  Q=註記
     R=術前診斷  S=預計心導管  T=每日續等清單
     ─ 註記：從入院提示欄擷取（如「3/30無床延期」）
     ─ 每日續等清單：你填 1 = 標記要搬到續等清單

  【續等清單整合】
  你的操作：
  - 「整合續等」或「續等清單合併進入院序」

  Claude 會：
  1. 讀取每天續等清單（A-I 欄）
  2. 判斷續等病人的主治醫師是否已在當日入院序有排程（有隔日導管時段）
     ─ 有時段 → 接在 round-robin 之後繼續排
     ─ 無時段 → 排在最後
  3. T 欄標記「1」給續等的病人
  4. 備註欄加「續等/原備註」前綴以區分

  【重要】寫入 Sheet 前先讀取目標區域，確認為空才寫入，不覆蓋現有資料

--------------------------------------------
步驟 4.2：抽籤每日續等主治醫師入院順位
--------------------------------------------
  你的操作：
  - 「抽每日續等」

  Claude 會：
  - 每位醫師平等一支籤
  - 結果貼在每天續等清單的 H-I-J 欄

--------------------------------------------
步驟 4.5：整合入院清單（產生定案）
--------------------------------------------
  你的操作：
  - 「整合定案」或「產生定案」

  Claude 會：
  - 建立「YYYYMMDD 定案」工作表
  - 上半部=當日入院清單（淺綠底）
  - 下半部=每日續等清單（粉橘底）
  - 序號連續編號

--------------------------------------------
步驟六：心導管排程 key-in [skill: admission-cathlab-keyin]
--------------------------------------------
  系統：http://cardiopacs01.hosp.ncku:8080/WEBCVIS/HCO/HCO1W001.do
  帳號/密碼：107614 / 107614

  你的操作：
  - 「導管排程」或「key-in 導管」
  - 沒有填術前診斷/預計心導管的病人（無資料病人）→ 照樣 key 入排程
    照主治醫師時段排入，術前診斷/預計心導管留空，備註填「無資料病人」
  - 備註寫「不排程」「4/3已做 不排程」等字眼的病人 → 直接跳過不排

  Claude 會：
  1. 設定日期為導管日（入院隔天）
  2. 先擷取並保存現有排程資料
  3. 比對病歷號，跳過已存在的病人
  4. 依主治醫師時段表填入：
     - 檢查室（H1/H2/C1/C2）
     - 時間（上午 0600+/下午 1730+/非時段 H1 1800+）
     - 主刀醫師（attendingdoctor1）
     - 第二醫師（attendingdoctor2）→ 只限括號簡稱對應的五位：
       寬→葉建寬  浩→葉立浩  晨→洪晨惠  齡→許毓軨  嘉→蘇奕嘉
     - 術前診斷（依 Excel F 欄）
     - 預計心導管（依 Excel G 欄）
  5. 完成後重新查詢，比對舊資料確認沒被洗掉

  【安全規則】
  - 絕對不更動現有排程 — 只新增，不修改、不刪除
  - 已存在的病人不重複新增
  - 主治醫師欄位（主刀/第二主刀）不填入李柏增

============================================
  相關工作表說明
============================================

  每天續等清單：
    欄位（A-E）：主治醫師、姓名、病歷號、EMR、EMR摘要
    抽籤結果（H-I-J）：序號、主治醫師、病人姓名

  主治醫師時段表：
    門診時段詳細表（含診間與檢查室對應）
    Col C=週一 D=週二 E=週三 F=週四 G=週五
    上午(H1,H2,C1,C2) → 下午(H1,H2,C1,C2)
    括號簡稱=第二主刀醫師（只限五位：寬/浩/晨/齡/嘉）

  主治醫師抽籤表（第 3 個 sheet）：
    A=星期一 ~ E=星期五、Row 1=header、*2=兩支籤

  下拉選單：
    A 欄=術前診斷（65 項，與導管排程網站同步）
    D 欄=預計心導管（22 項，與導管排程網站同步）

  YYYYMMDD：
    A-L = 病人主資料（12 欄）
    N-T = 入院序列表格（7 欄：序號、主治醫師、病人姓名、
          註記、術前診斷、預計心導管、每日續等清單）
          T 欄填 1 → 標記加入每天續等清單，等你指示後搬移
    資料下方 = 各醫師病人表格（8 欄 A-H）
      A=姓名  B=病歷號  C=EMR  D=EMR摘要
      E=手動設定入院序  F=術前診斷(下拉)  G=預計心導管(下拉)  H=註記

  YYYYMMDD 定案：
    最終入院順序（當日 + 續等合併）
