============================================
  第一階段：匯入與抽籤
============================================

--------------------------------------------
步驟一：匯入病人資料 [skill: admission-image-to-excel]
--------------------------------------------
  你的操作：
  - 直接將入院名單截圖貼給 Claude
  - 可一次貼多天的截圖
  - 範例指令：「這是 20260407-09 的三天預約入院清單，請幫我整理」

  Claude 會：
  1. 辨識圖片中的病人資料（12 欄：A-L）
  2. 自動比對主治醫師正確名單，修正 OCR 誤讀
  3. 自動建立日期工作表，寫入 A-L 欄
     ─ Header 列藍底（BDD7EE）粗體、資料列白底、加細框線
     ─ 全部儲存格靠左對齊
     ─ 標準欄寬 A-L: 100/100/90/80/70/189/160/40/90/50/120/40
       （G=160 是為了子表格的「預計心導管」內容能完整顯示）
  4. 列出所有病歷號請你確認
     （OCR 讀取數字容易出錯，務必逐筆核對！）

  注意：
  - 截斷的姓名不需要補全，步驟三 EMR 會自動校正
  - 病歷號固定 8 碼，以文字存入（保留前導零）
  - 低解析度截圖會先放大 3-6 倍分區裁切再辨識

  【差異更新模式】（同日期工作表已存在時）
  入院清單在實際入院日前會變動，user 會在臨近日重新發新的截圖。
  此時 Claude 必須做差異更新，而非整張覆蓋：
  1. 解析新截圖 → 與現有 sheet 用「病歷號」做 diff
  2. 列出：新增 N 人 / 取消 M 人 / 保留 X 人 → 等 user 確認
  3. 新增 → append 主表 + 醫師子表 + 跑 EMR + 預填 F/G
  4. 取消 → 刪主表 row + 子表 row + N-V 入院序
  5. 保留 → 完全不動（EMR/F/G/入院序/註記都保留）
  6. 更新醫師人數標題（如「劉嚴文（2人）」→「劉嚴文（3人）」）
  7. 重套統一格式（藍標題/白底/邊框/靠左/F/G dropdown/空行白底）
  8. 若 N-V 已手動定案，先問 user 是否要重跑入院序
  9. 病人換醫師（同病歷號不同醫師）視為「舊醫師取消 + 新醫師新增」

--------------------------------------------
步驟二：主治醫師抽籤排序 [skill: admission-lottery]
--------------------------------------------
  你的操作：
  - 「抽籤」或「排入院順序」
  - 指定醫師優先：「抽籤，陳則瑋排第一」
  - 特殊排序：「無條件讓 XXXXXXXX 第一個入院，備註員工家屬」

  Claude 會：
  1. 判斷入院日 → 查隔天門診醫師（主治醫師抽籤表）
     日→一、一→二、二→三、三→四、四→五
     （五/六需你手動指定）
  2. 區分醫師：
     ─ 在抽籤表 → 參加主抽籤
     ─ 不在抽籤表 → 仍留在工作表，但排在主抽籤之後
  3. 隨機抽籤，*2 醫師兩支籤
  4. 產生 round-robin（N-P 欄：序號、主治醫師、病人姓名）
  5. 在主資料下方排列各醫師病人表格（8 欄 A-H）
     A=姓名  B=病歷號  C=EMR  D=EMR摘要
     E=手動設定入院序  F=術前診斷  G=預計心導管  H=註記
     ─ F、G 欄有下拉選單（與導管排程網站選項同步）
     ─ 整理表起始列與萃取資料相隔至少 2 行空白
     ─ 【格式統一】每個醫師區塊都要套用完整格式：
        • 醫師標題列 A:H 合併、藍底（BDD7EE）、粗體
        • 副標題列藍底、粗體
        • 病人資料列白底
        • 整個區塊加細框線
        • F/G 欄都要有下拉選單
        • 全部儲存格靠左對齊
        • 不只第一個區塊，所有區塊都要！（4/14-16 曾出現只有第一個區塊有格式的 bug）

  【重要：非抽籤表醫師的處理】（已被修正多次，務必遵守！）
  - 所有醫師的病人都留在當日工作表，不會被移走
  - 不在抽籤表的醫師絕對不能放入主抽籤的 round-robin
  - 非時段醫師的序號接在有時段醫師最後一位病人之後

  【Round-Robin 排法（兩段獨立 RR — 重要！）】
  時段組（在當日抽籤表內的醫師）和非時段組（不在）
  分別跑 round-robin，**絕對不要把兩組混在一起 RR**：

  Step 1: 時段組彼此抽籤決定組內順序 → round-robin **排完所有時段組病人**
          A1→B1→C1→A2→C2→...（時段醫師之間真 RR，醫師用完就 skip）
  Step 2: 非時段組彼此抽籤決定組內順序 → 接在時段組**之後**，自己再 round-robin
          D1→E1→D2→...

  ❌ 錯誤示範（5 醫師全部一起 RR）：
     1. 黃鼎鈞-X (時)
     2. 廖瑀-Y  (時)
     3. 黃睦翔-Z (非) ← 錯！非時段不該插在時段中間
     4. 詹世鴻-W (時)
     ...
  ✅ 正確：時段醫師全部排完 → 非時段醫師接後面

  【組內醫師 lottery 算法】
  必須跑真正的 random.shuffle()（不可借用 sub-table 的醫師出現順序）：
    1. 從 主治醫師抽籤表 對應星期欄讀 col_values
    2. 建 weighted pool：純名 = 1 次, *2 = 2 次（兩支籤）
    3. random.shuffle(pool) + first-appear dedupe → 抽籤順序
    4. Filter to admit_doctors → 時段組
    5. admit_doctors - pool = 非時段組（也要 random.shuffle）

  【組內病人 lottery】
  多病人醫師（如 詹世鴻 3 人）內部順序也要 random.shuffle()
  使用者若手動指定（在子表格 E 欄填 1/2/3）就照使用者順序。

--------------------------------------------
步驟三：自動提取 EMR 病歷摘要 [skill: admission-emr-extraction]
--------------------------------------------
  前提：Chrome 已開啟 Web EMR 並登入
  （抽籤完成後 Claude 會自動開始，不需額外指令）

  Claude 會：
  1. 自動批次查詢所有病人（約 2-3 秒/人）
  2. 自動校正姓名/年齡/性別（從 EMR #divUserSpec 讀取）：
     - 只要病歷號輸入正確，#divUserSpec 就會回傳該病人的姓名/生日/性別，
       不管最後是否點到本醫師門診，都會用這份資料更新 Sheet
     - 姓名：更新 Sheet 主資料 F 欄 + 子表格 A 欄
     - 年齡：由生日推算至實際住院日（未滿生日 -1），寫入主資料 H 欄
     - 性別：寫入主資料 G 欄
     - 不需詢問使用者
  3. 生成四段式摘要：
     一、心臟科相關診斷
     二、病史
     三、客觀檢查
     四、本次住院計畫
  4. 寫入醫師病人表格 C/D 欄
     C 欄=完整 SOAP（門診 header + [Diagnosis][Subjective][Objective][Assessment & Plan]），
          截斷於 [Medicine]，不含藥品/檢驗/掛號
     D 欄=四段式摘要（內容也從截斷過的 SOAP 取得，同樣不含藥品之後的資料）
  5. 無紀錄 → C 欄寫「無本院一年內主治醫師門診紀錄」
  6. 主治醫師無門診紀錄時 → 只能 fallback 到以下六位白名單醫師：
     劉秉彥、趙庭興、蔡惟全、許志新、陳柏升、李貽恒
     （例：20260420 戴宗訓主治是陳則瑋，實際由劉秉彥看診 → 採用）
     白名單外的科別（如眼科、骨科）即使同一病人也**不採用**，
     因為 SOAP note 沒有心臟相關內容會污染摘要 → 翻成無資料病人
     （C 欄寫「無本院一年內主治醫師門診紀錄」），
     但姓名/年齡/性別仍會從 #divUserSpec 校正回 Sheet

  技術補充：
  - 本地 HTTP server (port 18234) 或 Playwright 帶 session URL 兩種抓取方式
  - Playwright 方式：user 先手動登入後貼 session URL（含 (S(xxx))），
    Claude 用該 URL 導航並在 topFrame/leftFrame/mainFrame 中互動
  - leftFrame 載入較慢，需 `wait_for_function` 直到 `<a>` 出現（至少 3 秒）
  - 首次批次完成後會重試載入失敗的病人
  - EMR 切斷規則（兩欄都要套用）：遇到 `[Medicine]`、`[Plan : 依類別]`、
    `-------藥品-------`、`-------檢驗-------`、`-------其他-------` 就截斷
  - 不要 print 含 \xa0 的字串到終端（cp950 會 crash），只寫 UTF-8 log 檔

--------------------------------------------
步驟四：設定入院順序與下拉選單 [skill: admission-ordering]
--------------------------------------------
  【重要流程！Claude 必須遵守的順序】
  1. 抽籤 → 顯示結果（步驟二）
  2. 自動開始 EMR 摘要擷取（步驟三）
  3. EMR 完成後 → 等你手動在醫師病人表格的
     F 欄（術前診斷）和 G 欄（預計心導管）拉下拉選單
  4. 你確認完下拉選單後 → Claude 才一併把
     抽籤結果 + 術前診斷 + 預計心導管
     寫入 N-V 入院序列表格

  *** Claude 絕對不能在你拉完下拉選單之前寫入 N-V 欄 ***

  你的操作：
  - EMR 完成後，在 Sheet 的醫師病人表格 F/G 欄選擇下拉選單
  - 如果病人不需要導管排程，F/G 欄留空即可
  - 選完後告知 Claude：「下拉選單填好了，幫我整合入院序」
  - 或直接說「自動生成入院序」（Claude 會讀取現有 F/G 欄資料）

  Claude 會：
  1. 再次確認所有醫師是否在隔天抽籤表上
  2. 如有 2 位以上病人的醫師，檢查 E 欄是否已設定順序
  3. 一次寫入 N-V 欄（9 欄）：

     N=序號  O=主治醫師  P=病人姓名  Q=備註(住服)  R=備註
     S=病歷號  T=術前診斷  U=預計心導管  V=改期

     ─ 備註(住服)：留空，由住院服務中心填寫
     ─ 備註：從醫師表格 H 欄（註記）擷取（如「W1.3.5洗腎」「3/11無床延期」）
     ─ 病歷號：以文字格式儲存（保留前導零）
     ─ 術前診斷：只用子項目名稱（如 pAf），不需母清單前綴
     ─ 改期：你填 YYYYMMDD = 純人工註記，表示此病人不做 N+1 cathlab（另行處理）

步驟六：心導管排程 key-in [skill: admission-cathlab-keyin]
--------------------------------------------
  系統：http://cardiopacs01.hosp.ncku:8080/WEBCVIS/HCO/HCO1W001.do
  帳號/密碼：107614 / 107614

  Driver：python cathlab_keyin.py cathlab_patients_YYYYMMDD.json
        （2026-04-26 起改用通用 driver，舊的 cathlab_keyin_MMDD.py 廢止）

  你的操作：
  - 「導管排程」或「key-in 導管」
  - 備註欄寫「不排程」「檢查」的病人 → 從 JSON 排除
  - 沒有填術前診斷/預計心導管的病人 → 仍然要 key，照主治醫師時段排
  - V 欄有值（改期）的 ordering row → 從 JSON 排除

  Claude 會：
  0. **病人清單來源：子表格（統整資料，主治醫師病人清單）**，不是 N-V 入院序
     （N-V 是給住服的子集；子表格才有所有需要做 cathlab 的病人）
  1. 整理成 cathlab_patients_YYYYMMDD.json（每病人 cathlab_date / name / chart /
     doctor / second / room / time / diagnosis / procedure / note）
  2. 跑 python cathlab_keyin.py <json>，driver 自動：
     - 設定日期為導管日（入院隔天；週五入院 → 同日；張獻元週三入院 → 同日 PM）
     - WEBCVIS 登入 + 兩階段（Phase 1 ADD / Phase 2 UPT 補 pdijson/phcjson）
     - 病歷號 dedupe → 已存在自動 SKIP ADD（仍跑 UPT 補 F/G）
     - PDI/PHC ID 從 cathlab_id_maps.json 載入（66 diag + 22 proc 已驗證；
       新術式/診斷 ID 缺 → 報錯，使用者去 WEBCVIS 查值補進 json，不要自編）
  4. 依主治醫師時段表填入：
     - 檢查室（H1/H2/C1/C2）
     - 時間（有時段 AM 0600+/有時段 PM 1800+/無時段 H1 2100+）
     - 主刀醫師（attendingdoctor1）
     - 第二醫師（attendingdoctor2）→ 只限括號簡稱對應的五位：
       寬→葉建寬  浩→葉立浩  晨→洪晨惠  齡→許毓軨  嘉→蘇奕嘉
     - 術前診斷（依 Sheet F 欄）
     - 預計心導管（依 Sheet G 欄）
       若不在 WEBCVIS 下拉選單 → 改填入 note 備註欄，
       預計心導管欄位維持空白
       * 例外：TAVI 在 WEBCVIS 下拉選單中，直接填入（不走 note fallback）
         → 檢查室選「開刀房」（xa-外科開刀房25房），不是 H1/H2/C1/C2
  5. 完成後重新查詢，比對舊資料確認沒被洗掉

  【安全規則】
  - 絕對不更動現有排程 — 只新增，不修改、不刪除
  - 已存在的病人不重複新增
  - 主治醫師欄位（主刀/第二主刀）不填入李柏增（2026/08 前；2026/08 後需重新確認）

  【張獻元週二病人規則】
  張獻元的病人，週二入院時：
  - 註記欄（H）無「王思翰」「張倉惟」字樣 → 週二下午同日做導管（張獻元自己的週二時段）
  - 註記含「王思翰」或「張倉惟」→ 週三（N+1）用張獻元的週三導管時段做
    （王思翰/張倉惟 沒有獨立時段，是借張獻元的時段）
  - 兩種情境都 key 在「attendingdoctor=張獻元」，只差時段/日期

  【張獻元週三病人規則】
  張獻元的病人，週三入院時：
  - 週三同日做導管（張獻元週三 AM C2 或 PM C2 時段）
  - 週四張獻元無時段 → 不走 N+1 規則
  - key 在「attendingdoctor=張獻元」，cathlab 日=入院日同日

  【詹世鴻週五病人規則】（2026-04-17 修正）
  詹世鴻的病人，週五入院時：
  - 抽籤/入院序：詹世鴻視為「非時段醫師」，不進主 round-robin，排在無時段醫師後
  - 導管排程：排非時段（H1 2100+, note="本日無時段"），同日週五
  - lottery 與 cathlab 一致，兩邊都當非時段醫師處理
    （詹世鴻有週五 AM C2 時段，但入院病人不走該時段）

  【時段表括號用途】
  1. 同格內括號（如「劉嚴文(寬)」「陳昭佑(李柏增)」）→ 括號內是第二主刀
  2. 獨立列整格只有括號名（如單獨一列寫「(陳則瑋)」）→ 該醫師也是當天此時段的
     並列主刀，照正常時段排（AM 0600+ / PM 1800+），不要誤排「本日無時段」
     例：週二 PM H1 = 劉秉彥 + (陳則瑋) 並列

--------------------------------------------
改期註記（V 欄）
--------------------------------------------
  V 欄填 YYYYMMDD 純屬人工註記，表示該病人不做原訂 N+1 cathlab
  （另行處理，Claude 不自動搬 sheet / 不自動 DEL/ADD WEBCVIS）。
  - cathlab keyin 與 verify_cathlab.py 會把有 V 值的 ordering row 視為 skip
  - 手動寫 V 必須透過 P 欄姓名 / S 欄病歷號比對 ordering row
    （round-robin 打散後 row 順序與主資料不同，不能直接用主資料 row 編號）
